# Code Review Command

コードの品質、セキュリティ、パフォーマンスをレビューします。

## レビュー観点

### 1. セキュリティ 🔒
- [ ] XSS脆弱性がないか
- [ ] CSRF対策がされているか
- [ ] 認証・認可が適切か
- [ ] 機密情報がハードコードされていないか
- [ ] SQLインジェクション対策（Supabase RLS）
- [ ] 入力バリデーション

### 2. パフォーマンス ⚡
- [ ] 不要なレンダリングがないか
- [ ] メモ化が適切か（useMemo, useCallback）
- [ ] バンドルサイズは適切か
- [ ] 画像最適化されているか
- [ ] 遅延読み込みが適用されているか

### 3. コード品質 📝
- [ ] 型定義が適切か
- [ ] コンポーネントの責務が明確か
- [ ] DRY原則に従っているか
- [ ] 命名が適切か
- [ ] コメントが必要な箇所にあるか

### 4. アクセシビリティ ♿
- [ ] セマンティックHTMLを使用しているか
- [ ] aria属性が適切か
- [ ] キーボードナビゲーション対応
- [ ] カラーコントラスト

## 使用方法

```
/review [ファイルパス or 機能名]
```

## 出力形式

```markdown
## レビュー結果

### 問題点 (Critical/Warning/Info)
- ...

### 改善提案
- ...

### 良い点
- ...
```
